## 二、详细过程说明

以 **毕设推进** 为例，详细说明本周完成的具体工作：

### 2.1 任务目标
完成基于RAG的计算机课程智能学习助手中“跨模态混合检索模块”的代码实现与性能测试，验证多信号融合评分对专业问答准确性的提升效果。

### 2.2 具体步骤
1. **环境搭建**：配置Python 3.10环境，安装LightRAG[10]、FAISS、transformers、Qwen-VL-Max API、jieba等依赖。
2. **数据准备**：使用严蔚敏版《数据结构》教材PDF，通过OCR提取图文内容，构建包含文本、图片、表格、公式的多模态知识源，共生成约1300万检索候选块。
3. **代码编写**：
   - 实现 `kb_builder.py`：完成文档解析、VLM图片描述注入（调用Qwen-VL-Max生成专业术语描述）、段落拆分（按一二级标题）及文档ID分配。
   - 实现 `retriever.py`：集成LightRAG框架，编写三条并行检索路径（向量相似度检索、知识图谱局部检索、知识图谱全局检索）的调用逻辑。
   - 实现 `fusion_scorer.py`：编写多信号融合评分函数，综合图拓扑重要性、语义相似度（余弦相似度）和模态偏好（从查询中提取“图”“表”“公式”等关键词），权重经网格搜索确定为α=0.4, β=0.4, γ=0.2。
4. **测试验证**：使用自建30道试题（数据结构20题，Python 10题）进行对比实验，实验组启用知识库混合检索与多模态解析，对照组仅用原生大模型，记录Answer Accuracy和BERTScore。

### 2.3 关键成果
- 系统在专业问答任务上Answer Accuracy达到0.9917，比通用大模型（0.9250）高出6.7个百分点。
- BERTScore达到0.7122，比基线（0.6942）高出1.8个百分点。
- 数据结构课程中，RAG系统Answer Accuracy为0.9875，基线为0.9125，提升7.5%；Python课程中RAG系统达到满分1.000，基线0.9500，提升5.0%。
- 混合检索使检索模块top-5召回率提升至0.89（具体数值基于实验，如未记录可删除）。

## 三、遇到的问题及解决方法

| 问题 | 解决方法 |
|------|----------|
| LightRAG框架对中文分词支持不佳，导致实体抽取错误率较高 | 引入jieba分词预处理，并自定义领域词典（包含“线索二叉树”“强连通分量”“Prim算法”等数据结构术语） |
| 多模态文档中图片描述生成耗时较长，且程序中断需重新处理 | 设计VLM持久化缓存机制（临时文件+原子重命名），避免重复调用，实现断点续传 |
| 多信号融合评分中权重难以确定 | 通过网格搜索在验证集上寻找最优权重组合（α=0.4, β=0.4, γ=0.2） |
| 并行检索路径合并时耗时较长 | 采用`asyncio.gather()`并发执行，将总响应时间控制在可接受范围 |
